## 1. 시스템 개요

### 1.1 목적
사장님신용대출의 **전략등급별 금리(마진율)를 조정했을 때** 발생하는 다음 효과를 한 화면에서 시뮬레이션하고,
실제 금리 변경 시행 후의 **실적 변화를 추적**하기 위한 대시보드.

1. 신규유입 변화(즉시효과): 신규취급액, 순증액, 대출잔액(전체), 대출건수, 예상 신규고객 유입
2. 손익 변화(균형추정 — 월간 기준): 이자수익, 공헌이익, 영업이익
3. 간접효과: 카드발급고객, 수신 평잔 증가, MAU 고객 증가
4. 시행 실적: 과거 3건의 금리 변경 이벤트(3/31, 4/23, 5/29)에 대한 전/후 비교 (주간 영업지표, 월간 수익성, 퍼널 전환율)

### 1.2 기술 스택 / 구현 방식
- **단일 HTML 파일**로 구현 (배포·공유 용이성 우선, 별도 빌드 과정 없음)
- 파일 크기: **84KB** (v1.0 기준 — xlsx 지원 SheetJS 881KB 제거 후)
- `importmap` + esm.sh CDN을 통해 React 18.3.1 / ReactDOM 18.3.1 / Recharts 2.12.7 / lucide-react 0.383.0 로드
- `<script type="text/babel" data-presets="react" data-type="module">` — Babel Standalone으로 브라우저 내 JSX 인라인 트랜스파일
- 차트: Recharts (`BarChart`, `Bar`, `LabelList`, `ResponsiveContainer`, `Tooltip`, `Legend`, `ReferenceLine`, `CartesianGrid`, `XAxis`, `YAxis`)
- 아이콘: lucide-react
- 상태관리: React `useState` / `useMemo`
- **데이터 입력**: 모든 외부 데이터는 **CSV 파일 업로드**를 통해 클라이언트 사이드에서 파싱(FileReader)
- **CSV 다중 인코딩 자동 감지**: UTF-8 BOM → utf-8 / UTF-16 LE BOM → utf-16le / BOM 없음 → EUC-KR 시도 후 UTF-8 폴백

> **CDN 의존성 주의**: 파일 오픈 시 인터넷(esm.sh) 접근이 필요합니다. 행내 PC + 행내 와이파이 환경에서는 정상 작동하며, 오프라인 또는 인터넷 차단 환경에서는 빈 화면이 됩니다. 보고 전날 PDF 백업을 권장합니다.

##### (4) "신규유입 변화" 섹션 — 즉시효과

| 라벨 | 단위 | 산출 | 비고 |
|---|---|---|---|
| 신규취급액 (주) | 억 | `newWkOld → newWkNew` (소수점 1자리) | — |
| 순증액 (주) | 억 | `netWkOld → netWkNew` (소수점 1자리) | — |
| **대출 잔액 (총합)** | 억 | `(balOld+165.2) → (balNew+165.2)` (소수점 1자리) | 전략 1~7등급 + 레거시 8~10등급 구분 2줄 표기 |
| 대출 건수 (주간) | 건 | `cntOld → cntNew` (정수, 콤마) | — |
| **예상 신규고객 유입(주간)** | 명 | `custGainOld → custGain` (정수, 콤마) | 하단에 "3개월 평균 당행 최초 등록고객 비율 반영" 표시 |

- `custGainOld = Σ(cntOld × newCustRatio/100)` / `custGain = Σ(cntNew × newCustRatio/100)` (절대 수치, before/after 비교)
- `newCustRatio`: 해당 등급 실행 고객 중 당행 최초 등록 고객 비율(3개월 평균 **실측값**)
- **`churnNew` 산출에 쓰이는 `elasticity`가 1~4등급은 실측값(§4.8)** — 잔액·건수·취급액 변화 추정치의 정확도가 이번 개선으로 향상됨

##### (5) "추정 손익" 섹션 — 균형추정(월간 기준)

| 라벨 | 단위 | 산출 |
|---|---|---|
| 이자수익 (월) | 백만원 | `toMonthlyMM(intOld) → toMonthlyMM(intNew)` |
| 공헌이익 (월) | 백만원 | `toMonthlyMM(cmOld) → toMonthlyMM(cmNew)` |
| 영업이익 (월) | 백만원 | `toMonthlyMM(opOld) → toMonthlyMM(opNew)` |

- 미니노트: "추정손익은 변경된 조정금리가 지속되어 포트폴리오 내 모든 고객이 새 조정금리를 적용받는 균형점에 도달했을 때의 추정치입니다"

## 4. 계산 로직 정의

### 4.1 공통 유틸리티
| 함수 | 정의 |
|---|---|
| `avg(rows, key)` | 배열 평균 (빈 배열→0) |
| `sumBy(rows, key)` | 배열 합계 |
| `toMonthlyMM(v)` | `v × 100 / 12` — 연간(억) → 월간(백만원) |

### 4.2 `computeRows(adjMap, base, cofVal)` — 등급별 시뮬레이션

> **부호 규칙**: `legalCostRate`, `contractCreditRate`, `contractOpRate`, `creditGapRate`, `opGapRate` 5종은 CSV에서 음수(−) 입력. `nonIntRate`·`newCustRatio`는 양수(+) 입력.

```
churnNew    = max(0, churn + (bp/10) × elasticity)
retainRatio = (100 − churnNew) / (100 − churn)
balNew      = balance × retainRatio

marginNew   = margin + bp/100
intOld      = balance × margin / 100
intNew      = balNew  × marginNew / 100

// 음수 입력값 → + 처리로 차감 효과 동일
cmRateOld   = margin    − cof + legalCostRate + contractCreditRate
cmRateNew   = marginNew − cof + legalCostRate + contractCreditRate
cmOld/New   = balance(balNew) × cmRate / 100

// 음수 입력값 → − 처리로 양수 비용 전환
opAdjRate   = −contractOpRate − creditGapRate − opGapRate − nonIntRate
opOld/New   = cmOld/New − balance(balNew) × opAdjRate / 100

cntNew = count × retainRatio  /  newWkNew = newWk × retainRatio  /  netWkNew = netWk × retainRatio
newCustRatio: 해당 등급 실측 신규고객 비율 — computeRows 반환값에 포함
```

> **잔액 산출 방식에 대한 참고**: `balNew`는 신규취급/상환을 흐름(flow)으로 누적하는 방식이 아니라, **이탈율 변화 비율(`retainRatio`)을 현재 잔액에 곱하는 방식**입니다. `bp=0`이면 `churnNew=churn`이 되어 `retainRatio=1`(잔액 불변). 따라서 `elasticity`의 정확도가 잔액·건수·취급액 추정치 전체의 정확도를 좌우합니다 — 1~4등급의 elasticity가 실측값으로 바뀐 것(§4.8)이 이 산식의 신뢰도를 직접 개선합니다.

### 4.3 `runSimulation(adjMap, base, cofVal, indirectParams)`

```
// 신규고객 유입 수 (절대값)
custGainOld = Σ (r.cntOld × r.newCustRatio / 100)   // 현재 주간 신규고객 수
custGain    = Σ (r.cntNew × r.newCustRatio / 100)   // 조정 후 주간 신규고객 수

// 간접효과 — custGain(조정 후 절대 수치) 기준
cardEffect    = custGain × (cardRate / 100)
depositEffect = custGain × depositRate / 10000      // 억; 표시 시 ×100 → 백만원
mauEffect     = custGain × (mauRate / 100)

반환: { rows, tot, indirect: { custGainOld, custGain, cardEffect, depositEffect, mauEffect } }
```

### 4.4 `findOptimalAdj(base, cofVal, { churnCap=15, range=50, step=5 })`
- 등급별 grid search: 공헌이익 최대화 bp, 이탈률 증가 ≤ 15%p 제약
- cm 산식: `margin + bp/100 − cof + legalCostRate + contractCreditRate`

### 4.8 `elasticity`(탄력성 계수) 추정 방법론 — 요약 _(v2.0 신설)_

> 대시보드 코드에 내장된 함수가 아니라, **simbase CSV에 입력할 값을 산출하기 위한 오프라인 분석 절차**입니다. 전체 진단 과정(다중공선성 발견·보정)과 결과는 [`CSV_데이터_가이드.md`](./CSV_데이터_가이드.md) 부록에 상세 기술되어 있습니다.

**모형** (등급별 개별 OLS 회귀, 1~4등급만):
```
churn_i,t = β0 + β1·bp_i,t + β2·ret_t + ε_i,t
elasticity = β1 × 10   (시뮬레이터 산식의 "10bp당" 단위에 맞춤)
```
- `churn_i,t`: 등급 i, 주차 t의 주간 이탈율(%)
- `bp_i,t`: 주차 t 시점 등급 i의 누적 금리조정폭(bp) — 3개 이벤트 발효 시점마다 계단식 누적
- `ret_t`: 주가지수의 전주 대비 변동률(%) — 시장 요인 통제 변수 (최초엔 지수 레벨을 썼으나 `bp`와 상관계수 −0.91로 다중공선성이 심해 변동률로 교체, 상관계수 −0.10으로 해소)

**최종 결과**:

| 등급 | elasticity (10bp당) | R² |
|---|---|---|
| 1등급 | 2.15 | 0.19 |
| 2등급 | 1.44 | 0.16 |
| 3등급 | 0.50 | 0.08 |
| 4등급 | 0.91 | 0.14 |

**범위 제한**: 5~7등급은 일부 이벤트에서 bp=0(조정 미적용)으로 표본이 1개 이벤트뿐이라 신뢰도가 낮아 이번 추정 대상에서 제외(§7 Open Item #2). simbase CSV 반영은 사용자가 직접 수행(대시보드/Claude가 자동 반영하지 않음).
