# CSV 데이터 가이드 — 사장님신용대출 금리전략 대시보드

이 문서는 `standalone.html` 대시보드에 업로드하는 9개 CSV 템플릿의 **컬럼별 의미, 단위, 산출 기준, 대시보드 내 사용처**를 정리한 해설서입니다.
실측 데이터를 추출할 때 "어떤 정의의 값을 어디서 뽑아야 하는지" 판단하는 기준으로 사용하세요.

## 1. `template_2_simbase.csv` — 시뮬레이터 기준값 (전략등급별)

**용도**: "금리 조정 시뮬레이터" 탭의 등급별 슬라이더, 결과 테이블, 신규유입/손익 카드의 **출발점(현재 상태) 값**입니다. "기준값 CSV" 업로드 버튼으로 교체합니다.

> ⚠️ **이번에 수정한 항목**: 기존 파일의 7번째 컬럼명이 `interestRate`였으나, 코드가 읽는 이름은 `margin`입니다. 이름을 일치시키지 않으면 업로드 시 시뮬레이션 전체가 깨집니다(margin이 undefined → 모든 계산이 NaN). **`margin`으로 수정 완료**했습니다.

> 📐 **표시 정밀도 변경(2026-06-15)**: `balance`/`newWk`/`netWk`는 시뮬레이터의 "신규유입 변화" 카드와 등급별 결과 테이블에서 **소수점 1자리(n.n억)**까지 표시되도록 변경되었습니다. 데모 값도 소수 1자리로 갱신했으니, 실측 데이터도 가능하면 정수로 절사하지 말고 소수 1자리까지 입력하세요(정수만 입력해도 "620.0"처럼 표시되어 동작에는 문제없음).

> ⚠️ **비용율 입력 부호 규칙 (2026-06-19 변경)**:
> `legalCostRate`, `contractCreditRate`, `contractOpRate`, `creditGapRate`, `opGapRate` **5개 원가율 컬럼은 음수(−)로 입력**합니다.
> 예: `legalCostRate = -0.02`, `contractCreditRate = -0.10`, `contractOpRate = -0.18`
> `nonIntRate`(비이자이익율)는 이익 항목이므로 **양수(+)로 입력**합니다. 예: `nonIntRate = 0.10`
> 코드 내부에서 음수값을 역전하여 원가를 차감하는 방식으로 처리하므로, 부호를 반드시 지켜야 계산이 정확합니다.

| 컬럼 | 한글명 | 단위 | 정의 | 비고 |
|---|---|---|---|---|
| `grade` | 전략등급 | 텍스트 | 1등급~7등급 (고신용→중저신용 순) | 행 식별자, 7개 행 고정 |
| `balance` | 대출 잔액 | 억원 (소수 1자리 표시) | 해당 등급의 현재 사장님신용대출 총 잔액 | 시뮬레이션의 "현재" 출발 잔액 |
| `count` | 대출 건수 | 건 (주간 기준) | 해당 등급의 주간 대출 건수 | 신규유입 변화 카드 "대출 건수(주간)"의 현재값 |
| `newWk` | 신규취급액 (주간) | 억원/주 (소수 1자리 표시) | 해당 등급의 주간 신규 대출 실행액 | "신규유입 변화" 카드 |
| `netWk` | 순증액 (주간) | 억원/주 (소수 1자리 표시) | 해당 등급의 주간 순증액(실행−상환) | "신규유입 변화" 카드 |
| `churn` | 여정이탈률 | % | 대출 여정(한도·금리 확인) 중 최종 실행에 도달하지 못하는 고객 비율의 **현재값** | 금리 조정 시 `elasticity`와 함께 변화 |
| `margin` | 적용금리(마진율) | % | 해당 등급에 적용되는 평균 금리(연율) | `이자수익 = balance × margin / 100` (연간 억원) |
| `legalCostRate` | 법적비용율 | % | 연간 법적비용 원가율 | **음수(−) 입력** — 공헌이익 계산 시 `+ legalCostRate`로 가산(= 차감 효과) |
| `contractCreditRate` | 계약신용원가율 | % | 연간 계약 신용원가율 | **음수(−) 입력** — 공헌이익 계산 시 `+ contractCreditRate`로 가산(= 차감 효과) |
| `contractOpRate` | 계약업무원가율 | % | 연간 계약 업무원가율 | **음수(−) 입력** — 영업이익 계산 시 `−contractOpRate`로 부호 역전하여 차감 |
| `creditGapRate` | 신용원가 Gap율 | % | 계약원가와 실제신용원가의 차이(Gap) | **음수(−) 입력** — 영업이익 계산 시 `−creditGapRate`로 부호 역전하여 차감 |
| `opGapRate` | 업무원가 Gap율 | % | 계약원가와 실제업무원가의 차이(Gap) | **음수(−) 입력** — 영업이익 계산 시 `−opGapRate`로 부호 역전하여 차감 |
| `nonIntRate` | 비이자이익율 | % | 카드·예금 등 비이자수익 기여율 | **양수(+) 입력** — 영업이익 계산에서 가산(+) |
| `elasticity` (ε) | 금리 탄력성 계수 | 무차원 | 금리 10bp 변화 시 이탈률(churn)이 변하는 정도 | `churnNew = churn + (bp/10) × elasticity` |
| `newCustRatio` | 당행 최초 등록고객 비율 | % | 해당 등급 대출 실행 고객 중 당행에 기존 정보가 없던 **완전 신규고객 비율** (3개월 평균) | 시뮬레이터 "예상 신규고객 증가분(주간)" 카드 및 간접효과(카드발급·예금·MAU) 산출 분모. **양수(+)로 입력** (예: `62`) |

### 계산식 참고 (코드 `computeRows` 기준 — 2026-06-19 산식 기준)

> **부호 규칙 요약**: 원가율 5개(`legalCostRate` 등)는 CSV에서 음수로 입력됩니다.
> 코드는 이를 가산(`+`) 또는 부호 역전(`−`)하여 처리하므로, 경제적 의미(원가 차감)는 동일합니다.

> 📐 **잔액 산정방식 변경(2026-06-26)**: 기존엔 `balNew = balance × retainRatio`(이탈율 변화 비율을 현재 잔액에 곱하는 균형 스케일링 방식)였으나, **주간 순증액(`netWk`)이 12주(약 3개월) 동안 유지된다고 가정하고 누적하는 방식**으로 변경되었습니다. `balOld`도 더 이상 `balance` 원값이 아니라 "+3개월 후, 조정 없이 현재 추세가 유지될 때의 예상 잔액"으로 재정의되어, 이자수익·공헌이익·영업이익까지 모두 "+3개월 후 추정"으로 의미가 바뀌었습니다(화면에 **"+3M"** 표기 추가). `findOptimalAdj()`(최적금리 탐색)도 동일한 +3M 기준으로 갱신했습니다.
>
> 📐 **"최적금리 찾기" → "균형성장" 버튼 2종 분리(2026-06-26 추가 변경)**: `findOptimalAdj()`가 등급별 독립 탐색이 아니라 **1~7등급 포트폴리오 전체(합계) 기준**으로 동작하도록 변경했습니다. 등급마다 개별적으로 잔액·공헌이익이 둘 다 늘 필요는 없고, **일부 등급은 금리를 내려 잔액을 벌고 다른 등급은 올려서 공헌이익을 버는 식으로 서로 메꿔서, 7개 등급 합계가 둘 다 현재보다 성장(`Σ balNew > Σ balOld` AND `Σ cmNew > Σ cmOld`)**하면 인정됩니다(등급별 이탈율 증가 15%p 제약은 그대로 유지). 이 조합 탐색은 등급×bp 후보 조합 수가 너무 커서(21^7) 전체탐색이 불가능해, **동적계획법(DP)으로 정확한 최적해**를 찾습니다: "균형성장(공헌이익 위주)"는 `Σ잔액변화 ≥ 0` 제약 하에 `Σ공헌이익변화` 최대화, "균형성장(자산성장 위주)"는 `Σ공헌이익변화 ≥ 0` 제약 하에 `Σ잔액변화` 최대화. 등급별로 서로 다른(때로는 반대 방향의) bp가 나올 수 있으며, 둘 다 "참고용" — 최종 결정은 화면에서 직접 조정한 값을 사용하세요.

```
churnNew    = max(0, churn + (bp/10) × elasticity)
retainRatio = (100 − churnNew) / (100 − churn)
netWkNew    = netWk × retainRatio

// +3개월(12주) 예상 잔액 — 흐름(순증액) 누적 방식
balOld      = balance + netWk    × 12   // 조정 없이 현재 추세 유지 시
balNew      = balance + netWkNew × 12   // 조정 후 추세 반영 시

marginNew   = margin + bp/100
intOld      = balOld × margin   / 100                      // +3M 시점 연간 이자수익(억원)
intNew      = balNew × marginNew / 100

// legalCostRate, contractCreditRate는 CSV에서 음수 → + 하면 차감 효과
cmRateOld   = margin    − cof + legalCostRate + contractCreditRate
cmRateNew   = marginNew − cof + legalCostRate + contractCreditRate
cmOld/New   = balOld/balNew × cmRate / 100                  // +3M 시점 연간 공헌이익(억원)

// contractOpRate, creditGapRate, opGapRate는 음수 → −(음수) = 양수 비용으로 전환
// nonIntRate는 양수 → −nonIntRate로 비용합계에서 상쇄
opAdjRate   = −contractOpRate − creditGapRate − opGapRate − nonIntRate
opOld/New   = cmOld/New − balOld/balNew × opAdjRate / 100   // +3M 시점 연간 영업이익(억원)
```

**수식 검증 예시 (1등급 기준)**:
```
입력값: legalCostRate=-0.02, contractCreditRate=-0.10, contractOpRate=-0.18,
        creditGapRate=-0.01, opGapRate=-0.01, nonIntRate=0.10
cmRateOld = margin − cof + (−0.02) + (−0.10) = margin − cof − 0.12  ← 원가 0.12% 차감 ✓
opAdjRate = −(−0.18) − (−0.01) − (−0.01) − 0.10 = 0.18+0.01+0.01−0.10 = 0.10  ← 순비용 0.10% ✓
```

```
// 신규고객 유입 수 (절대값 — 조정 전/후 각각 산출)
// newCustRatio: 해당 등급 실행 고객 중 당행 최초 등록 고객 비율 (실측값, %)
custGainOld = Σ (cntOld × newCustRatio / 100)  // 현재 주간 신규고객 수 → 카드 "조정 전" 값
custGain    = Σ (cntNew × newCustRatio / 100)  // 조정 후 주간 신규고객 수 → 카드 "조정 후" 값 + 간접효과 분모
```

- "예상 신규고객 유입(주간)" 카드는 `custGainOld → custGain` 을 before/after로 표시합니다.
- 간접효과(카드발급·예금·MAU)는 `custGain`(조정 후 절대 수치) 기준으로 산출합니다.

- `cof`(자금원가율, 기본값 2.8%)는 CSV가 아니라 화면 상단 입력란에서 직접 입력합니다.
- 대시보드의 "이자수익(월)/공헌이익(월)/영업이익(월)" 카드는 위 연간(억원) 값을 `× 100 ÷ 12`하여 월간(백만원)으로 환산해 표시합니다.

### 데이터 추출 가이드
- `balance`, `newWk`, `netWk`, `count`: 사장님신용대출 등급별 최근 주간 영업 실적에서 추출
- `churn`: 퍼널 데이터(아래 4번 문서)에서 "가심사승인→실행 전환율"의 역수 개념(미전환율)으로 근사 가능, 또는 별도 여정 로그 분석 필요
- `margin`, `legalCostRate`, `contractCreditRate`, `contractOpRate`, `creditGapRate`, `opGapRate`, `nonIntRate`: 원가 모델/수익관리 부서 기준값
- `elasticity`: 과거 금리 변경 이벤트 전후 이탈률 변화량 ÷ (bp변화/10) 으로 역산 — 현재는 **가정값**이며 ADW 실측 데이터 확보 후 교체 예정
- `newCustRatio`: 여신 실행 고객 중 당행 최초 등록 고객 비율 — 직전 3개월 평균. 고객 DB 또는 CRM에서 등급별로 집계. 이 수치가 없으면 100(전원 신규)으로 입력 시 종전 방식(대출 건수 변화를 그대로 신규고객으로 간주)과 동일하게 작동

## 부록. `elasticity`(탄력성 계수) 추정 방법론 (2026-06)

`template_2_simbase.csv`의 `elasticity` 컬럼(1~4등급)은 가정값이 아니라 아래 방법으로 **실측 추정**한 값입니다. 산출 근거는 이 섹션을 참고하세요. (5~7등급은 표본 부족·노이즈로 이번 추정 대상에서 제외 — 향후 데이터 누적 후 별도 추정)

### 추정 모형
등급별로 개별 OLS(최소자승법) 회귀를 적용했습니다.

```
churn_i,t = β0 + β1·bp_i,t + β2·ret_t + ε_i,t
```

| 기호 | 정의 |
|---|---|
| `churn_i,t` | 등급 i, 주차 t의 주간 이탈율(%) — 종속변수 |
| `bp_i,t` | 주차 t 시점에 등급 i에 적용 중이던 누적 금리조정폭(bp). 이벤트(3/31, 4/23, 5/29) 발효 시점마다 계단식으로 누적 |
| `ret_t` | 주가지수의 전주 대비 변동률(%) — 시장 요인을 통제하기 위한 변수 |
| `β1` | 핵심 추정 대상. `elasticity = β1 × 10` (시뮬레이터 산식 `churnNew = churn + (bp/10)×elasticity`와 단위를 맞추기 위해 ×10) |
| `β2` | 주가지수 1%p 변동당 이탈율 변화 (통제 계수) |

### 다중공선성 진단 및 보정
최초 시도에서는 `ret_t` 대신 주가지수 **레벨(절대값)**을 사용했으나, `bp`와 지수 레벨의 상관계수가 **r ≈ −0.91**로 나타나 다중공선성이 심각했습니다(두 변수가 같은 6개월간 한 방향으로만 움직이는 추세라 회귀가 가격효과와 시장효과를 구분하지 못함 — 실제로 4개 등급 중 3개에서 부호가 반대로 나왔음). 지수를 **주간 변동률**로 교체하자 상관계수가 **r ≈ −0.10**으로 떨어져 문제가 해소되었고, 이후 4개 등급 모두 일관된(+) 부호를 얻었습니다.

### 최종 결과 (1~4등급)

| 등급 | elasticity (10bp당) | R² |
|---|---|---|
| 1등급 | 2.15 | 0.19 |
| 2등급 | 1.44 | 0.16 |
| 3등급 | 0.50 | 0.08 |
