## 1. 시스템 개요

### 1.1 목적
사장님신용대출의 **전략등급별 금리(마진율)를 조정했을 때** 발생하는 다음 효과를 한 화면에서 시뮬레이션하고,
실제 금리 변경 시행 후의 **실적 변화를 추적**하기 위한 대시보드.

1. 신규유입 변화(즉시효과) + **+3개월 예상 대출잔액**: 신규취급액, 순증액, 대출잔액, 대출건수, 예상 신규고객 유입
2. 손익 변화(+3개월 후 예상 — 월간 기준): 이자수익, 공헌이익, 영업이익
3. 간접효과(주간): 신규 카드발급 고객 증가, 수신평잔 증가효과, MAU 고객증가
4. 성과 모니터링: 과거 3건의 금리 변경 이벤트(3/31, 4/23, 5/29)에 대한 전/후 비교 (주간 영업지표, 월간 수익성, 퍼널 전환율)

### 1.2 기술 스택 / 구현 방식
- **단일 HTML 파일**로 구현 (배포·공유 용이성 우선, 별도 빌드 과정 없음)
- 파일 크기: **약 90KB**
- `importmap` + esm.sh CDN을 통해 React 18.3.1 / ReactDOM 18.3.1 / Recharts 2.12.7 / lucide-react 0.383.0 로드
- `<script type="text/babel" data-presets="react" data-type="module">` — Babel Standalone으로 브라우저 내 JSX 인라인 트랜스파일
- 차트: Recharts (`BarChart`, `Bar`, `LabelList`, `ResponsiveContainer`, `Tooltip`, `Legend`, `ReferenceLine`, `CartesianGrid`, `XAxis`, `YAxis`)
- 아이콘: lucide-react
- 상태관리: React `useState` / `useMemo`
- **데이터 입력**: 모든 외부 데이터는 **CSV 파일 업로드**를 통해 클라이언트 사이드에서 파싱(FileReader)
- **CSV 다중 인코딩 자동 감지**: UTF-8 BOM → utf-8 / UTF-16 LE BOM → utf-16le / BOM 없음 → EUC-KR 시도 후 UTF-8 폴백

> **CDN 의존성 주의**: 파일 오픈 시 인터넷(esm.sh) 접근이 필요합니다. 행내 PC + 행내 와이파이 환경에서는 정상 작동하며, 오프라인 또는 인터넷 차단 환경에서는 빈 화면이 됩니다. 보고 전날 PDF 백업을 권장합니다.

## 3. 화면별 요구사항

### 3.1 탭 1 — 금리 조정 시뮬레이터 (`Simulator`)

#### 3.1.1 목적
전략등급(1~7등급)별로 금리(마진율)를 ±50bp 범위에서 조정했을 때, 신규유입·이탈률·잔액·수익성·신규고객 유입에 미치는 영향을 "현재" 대비 "조정 후" 값으로 즉시 비교한다.

> **등급 범위 정책**: 신규 승인 대상은 **1~7등급**만. 8~10등급 레거시 잔액(165.2억)은 총 잔액 카드에만 구분 표기.

#### 3.1.2 화면 구성 (상→하 순서)

##### (1) 헤더 영역
- Gauge 아이콘 + "전략등급 금리 조정 시뮬레이터"
- 버튼 4종: **균형성장(공헌이익 위주)** / **균형성장(자산성장 위주)** / **초기화** / **기준값 CSV** (Upload)

##### (2) 안내 박스
- 아이콘: `Info`(참고용 안내)
- 문구: "탄력성 계수: 등급별 **실측 다중회귀분석** 기반(2026-06)"
- 핵심 산식:
  - **공헌이익 = 이자수익 − 자금원가율 + 법적비용 + 계약신용원가** _(비용율 CSV 음수(−) 입력)_
  - **영업이익 = 공헌이익 − 계약업무원가 − 신용원가gap − 업무원가gap + 비이자이익** _(비용율 CSV 음수(−) 입력)_
- CSV 헤더 스펙: `grade,balance,count,newWk,netWk,churn,margin,`**`legalCostRate,contractCreditRate,contractOpRate,creditGapRate,opGapRate,nonIntRate`**`,elasticity,`**`newCustRatio`**
- 현재 적용 파일명 표시 (`현재: {baseFile}`)
- **자금원가율(cof) 입력**: 기본값 **2.8%**, 즉시 반영

##### (3) 등급별 조정 슬라이더 _(v2.2 확정)_
- 1~7등급, 각 행: 슬라이더(`min=-50, max=50, step=5bp`) + 조정값 표시 + `ε {elasticity}` 표시
- 슬라이더는 **네이티브 `<input type="range">`를 그대로 사용**(파란색 채움 효과 유지를 위해 `accentColor` 외 별도 외형 재정의 없음)
- 슬라이더 트랙 바로 아래에 **5bp 단위 눈금**을 별도 레이어로 표시: **10bp 배수**(−50,−40,...,40,50)는 `C.faint` 색으로 굵고 길게, 그 사이 **5bp 단위**(−45,−35,...,35,45)는 `C.border` 색으로 얇고 짧게 — 슬라이더 자체 렌더링과는 분리된 장식 요소(`pointerEvents` 없음, 클릭/드래그에 관여하지 않음)

##### (4) "신규유입 변화" 섹션 — 즉시효과 + +3M 잔액

| 라벨 | 단위 | 산출 | 비고 |
|---|---|---|---|
| 신규취급액 (주) | 억 | `newWkOld → newWkNew` (소수점 1자리) | 당주 즉시효과 |
| 순증액 (주) | 억 | `netWkOld → netWkNew` (소수점 1자리) | 당주 즉시효과 |
| **+3M 예상 대출잔액 (총합)** | 억 | `(balOld+165.2) → (balNew+165.2)` (소수점 1자리) | `balOld/balNew`는 "현재 잔액 + 주간 순증액×12주" 누적 방식. 전략 1~7등급 + 레거시 8~10등급 구분 2줄 표기 |
| 대출 건수 (주간) | 건 | `cntOld → cntNew` (정수, 콤마) | 당주 즉시효과 |
| **예상 신규고객 유입(주간)** | 명 | `custGainOld → custGain` (정수, 콤마) | 당주 즉시효과. 하단에 "3개월 평균 당행 최초 등록고객 비율 반영" 표시 |

- `custGainOld = Σ(cntOld × newCustRatio/100)` / `custGain = Σ(cntNew × newCustRatio/100)` (절대 수치, before/after 비교)
- 미니노트에 "대출 잔액(총합)은 +3개월 예상치, 그 외 카드는 당주 즉시효과" 구분 안내 유지

##### (5) "추정 손익" 섹션 — +3개월 후 예상(월간 기준)

| 라벨 | 단위 | 산출 |
|---|---|---|
| 이자수익 (월) | 백만원 | `toMonthlyMM(intOld) → toMonthlyMM(intNew)` |
| 공헌이익 (월) | 백만원 | `toMonthlyMM(cmOld) → toMonthlyMM(cmNew)` |
| 영업이익 (월) | 백만원 | `toMonthlyMM(opOld) → toMonthlyMM(opNew)` |

- 부제: "(+3개월 후 예상 — 월간 기준)"
- 미니노트: "추정손익은 주간 순증액이 12주(약 3개월)간 유지되어 잔액이 그만큼 변화한다고 가정했을 때, 그 시점(+3개월 후)의 월간 손익 추정치입니다"

##### (6) 등급별 상세 결과 테이블
- 1~7등급, 가로 스크롤
- 컬럼: 등급 / 조정(bp) / 이탈율(%) / 건수(주) / 취급액(주,억) / 순증액(주,억) / **+3M 잔액(억)** / **+3M 이자수익(억)** / **+3M 공헌이익(억)** / **+3M 영업이익(억)**

##### (7) 요약 차트 ① — 주간 물량
- BarChart, 3개 카테고리: 신규실행액 / 대출건수 / 순증액 (1~7등급 기준, 당주 즉시효과)
- 막대 2계열: `curr`("현재", barInactive) / `adj`("조정 후", primary)
- `barSize=40`, `barCategoryGap="20%"`, 높이 280px, 막대 상단 LabelList

##### (8) 요약 차트 ② — +3M 월간 수익성
- 차트 제목에 "+3M" 표기
- BarChart, 2개 카테고리: 이자수익 / 공헌이익 (백만원/월)
- 막대 2계열: `curr` / `adj`, 높이 280px

##### (9) 간접효과 섹션
- **gainBanner**: "조정 후 주간 신규고객 **{N}명** 기준" (`custGain` 절대 수치, primary 색)
- IndirectCard 3종:

| 카드 | 라벨 | 입력 라벨 | 분모 | 산출 |
|---|---|---|---|---|
| 카드발급고객 증가 | "신규 카드발급 고객 증가 (주간)" | 유입 고객 × n% | `custGain` (절대) | `custGain × (cardRate/100)` 명, 기본 cardRate=35% |
| 수신평잔 증가 | "수신평잔 증가효과 (주간)" | 사장님 신용대출 고객 1개월 평잔 | `custGain` (절대) | `custGain × depositRate / 100` 백만원, 기본 depositRate=338만원(실측) |
| MAU 고객 증가 | "MAU 고객증가 (주간)" | 유입 고객 × n% | `custGain` (절대) | `custGain × (mauRate/100)` 명, **기본 mauRate=260%** |

- 미니노트: "**\* n 값은 사장님 대출 보유고객의 3개월 평균 실측 값이 입력되어 있으며, 수기 조정 가능함.**"

## 4. 계산 로직 정의

### 4.1 공통 유틸리티
| 함수 | 정의 |
|---|---|
| `avg(rows, key)` | 배열 평균 (빈 배열→0) |
| `sumBy(rows, key)` | 배열 합계 |
| `toMonthlyMM(v)` | `v × 100 / 12` — 연간(억) → 월간(백만원) |

### 4.2 `computeRows(adjMap, base, cofVal)` — 등급별 시뮬레이션

> **부호 규칙**: `legalCostRate`, `contractCreditRate`, `contractOpRate`, `creditGapRate`, `opGapRate` 5종은 CSV에서 음수(−) 입력. `nonIntRate`·`newCustRatio`는 양수(+) 입력.

```
churnNew    = max(0, churn + (bp/10) × elasticity)
retainRatio = (100 − churnNew) / (100 − churn)
netWkNew    = netWk × retainRatio

// +3개월(12주) 예상 잔액 — 주간 순증액 누적 방식
balOld      = balance + netWk    × 12   // 조정 없이 현재 추세 유지 시
balNew      = balance + netWkNew × 12   // 조정 후 추세 반영 시

marginNew   = margin + bp/100
intOld      = balOld × margin   / 100                       // +3M 시점 연간 이자수익(억원)
intNew      = balNew × marginNew / 100

cmRateOld   = margin    − cof + legalCostRate + contractCreditRate
cmRateNew   = marginNew − cof + legalCostRate + contractCreditRate
cmOld/New   = balOld/balNew × cmRate / 100                   // +3M 시점 연간 공헌이익(억원)

opAdjRate   = −contractOpRate − creditGapRate − opGapRate − nonIntRate
opOld/New   = cmOld/New − balOld/balNew × opAdjRate / 100    // +3M 시점 연간 영업이익(억원)

cntNew = count × retainRatio  /  newWkNew = newWk × retainRatio
newCustRatio: 해당 등급 실측 신규고객 비율 — computeRows 반환값에 포함
```

### 4.3 `runSimulation(adjMap, base, cofVal, indirectParams)`

```
custGainOld = Σ (r.cntOld × r.newCustRatio / 100)
custGain    = Σ (r.cntNew × r.newCustRatio / 100)

cardEffect    = custGain × (cardRate / 100)
depositEffect = custGain × depositRate / 10000
mauEffect     = custGain × (mauRate / 100)

반환: { rows, tot, indirect: { custGainOld, custGain, cardEffect, depositEffect, mauEffect } }
```

### 4.4 `findOptimalAdj(base, cofVal, { churnCap=15, range=50, step=5, objective })` — "균형성장" 포트폴리오 최적화 _(v2.2 전면 재설계)_

**문제 정의**: 등급별 bp 후보(−50~+50, 5bp 간격, `churnCap` 제약 적용)를 7개 등급에 대해 동시에 선택하여,
- **공헌이익 위주**: `Σ(balNew−balOld) ≥ 0` 제약 하에 `Σ(cmNew−cmOld)` 최대화
- **자산성장 위주**: `Σ(cmNew−cmOld) ≥ 0` 제약 하에 `Σ(balNew−balOld)` 최대화

등급×bp 후보 조합이 21^7(약 18억) 가지라 전체탐색은 불가능 → **동적계획법(DP, multi-choice knapsack 구조)**으로 정확한 최적해를 산출:

```
gradeCandidates = base.map(s => { bp별 dBal, dCm 산출 (computeRows와 동일 로직) })
dp = { 0: { score:0, exact:0, picks:{} } }
gradeCandidates.forEach(grade별로 dp 갱신)
best = dp 중 exact≥0 이고 score 최대인 상태
return best.picks   // { grade: bp, ... } 형태, 7개 등급 전부 포함
```

- **버튼 동작은 100% 브라우저 클라이언트 사이드**에서 실시간 계산. 실데이터 CSV 업로드 후 버튼 클릭만으로 즉시 최적해 산출.

### 4.5 ResultPage 집계
- 주간 KPI: `avg(before/after, key)` → delta%
- 월간 KPI: `avg(filtered by beforeMonths/afterMonths, key) × 100` (억→백만원)
- 통합 차트: 고신용 + 중저신용 합산 × 100

### 4.6 `LEGACY_BALANCE` 상수
```js
const LEGACY_BALANCE = 165.2; // 8~10등급 레거시 잔액(억) — 조정 범위 외
```
- 적용: Simulator "+3M 예상 대출잔액 (총합)" 카드 단 한 곳
- 시뮬레이션 반영 안 됨 (취급액/수익성 계산은 1~7등급만)
- 변경 시 HTML 내 상수 직접 수정

### 4.7 `weeklyAxisMax` — 주간 차트 공통 Y축 상한 (ResultPage)

적용 위치: ResultPage 주간 차트(§3.2.4 (7))의 `<YAxis domain={[0, weeklyAxisMax]} />`.

### 4.8 `elasticity`(탄력성 계수) 추정 방법론 — 요약

**모형** (등급별 개별 OLS 회귀, 1~4등급만): `churn_i,t = β0 + β1·bp_i,t + β2·ret_t + ε_i,t`, `elasticity = β1 × 10`.

**현행 적용값** (2026-06-30 기준, 6월4주 확정치, **대시보드 적용 중**):

| 등급 | elasticity (10bp당) | R² |
|---|---|---|
| 1등급 | 1.65 | 0.15 |
| 2등급 | 0.93 | 0.08 |
| 3등급 | 0.36 | 0.04 |
| 4등급 | 0.74 | 0.11 |

**28주 재추정 시도 결과 (2026-07-16, 7월2주까지, 미적용 — 보류)**: _(v2.3.1 신규)_

| 등급 | elasticity (28주) | R² |
|---|---|---|
| 1등급 | 0.15 | 0.001 |
| 2등급 | 0.09 | 0.004 |
| 3등급 | 0.28 | 0.026 |
| 4등급 | 0.47 | 0.098 |

> **보류 사유**: 7월1주·7월2주에 ev529 이후(bp −14bp 유지) 구간에서 이탈율이 1등급 28.6%·29.5%로 급등(6월 평균 23.1% 대비 +5~6%p). 동일 bp에서 이탈율이 오히려 역전되면서 회귀계수가 붕괴(R² 0.001~0.098)됨. 급등 원인(계절성, 7월 주가 하락 −7.6%, 기타)이 미분리 상태라 신뢰도 부족 → **현행값(6월4주 기준) 유지**. 2026-07-10 금리 인상 이벤트(ev710) 데이터 충분히 축적된 후 bp 양방향 변동이 확보되면 재추정 예정.
