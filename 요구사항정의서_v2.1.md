## 1. 시스템 개요

### 1.1 목적
사장님신용대출의 **전략등급별 금리(마진율)를 조정했을 때** 발생하는 다음 효과를 한 화면에서 시뮬레이션하고,
실제 금리 변경 시행 후의 **실적 변화를 추적**하기 위한 대시보드.

1. 신규유입 변화(즉시효과) + **+3개월 예상 대출잔액**: 신규취급액, 순증액, 대출잔액, 대출건수, 예상 신규고객 유입
2. 손익 변화(+3개월 후 예상 — 월간 기준): 이자수익, 공헌이익, 영업이익
3. 간접효과(주간): 신규 카드발급 고객 증가, 수신평잔 증가효과, MAU 고객증가
4. 시행 실적: 과거 3건의 금리 변경 이벤트(3/31, 4/23, 5/29)에 대한 전/후 비교 (주간 영업지표, 월간 수익성, 퍼널 전환율)

### 1.2 기술 스택 / 구현 방식
- **단일 HTML 파일**로 구현 (배포·공유 용이성 우선, 별도 빌드 과정 없음)
- 파일 크기: **86.7KB** (v2.0 대비 소폭 증가 — 라벨/캡션 텍스트 추가)
- `importmap` + esm.sh CDN을 통해 React 18.3.1 / ReactDOM 18.3.1 / Recharts 2.12.7 / lucide-react 0.383.0 로드
- `<script type="text/babel" data-presets="react" data-type="module">` — Babel Standalone으로 브라우저 내 JSX 인라인 트랜스파일
- 차트: Recharts (`BarChart`, `Bar`, `LabelList`, `ResponsiveContainer`, `Tooltip`, `Legend`, `ReferenceLine`, `CartesianGrid`, `XAxis`, `YAxis`)
- 아이콘: lucide-react
- 상태관리: React `useState` / `useMemo`
- **데이터 입력**: 모든 외부 데이터는 **CSV 파일 업로드**를 통해 클라이언트 사이드에서 파싱(FileReader)
- **CSV 다중 인코딩 자동 감지**: UTF-8 BOM → utf-8 / UTF-16 LE BOM → utf-16le / BOM 없음 → EUC-KR 시도 후 UTF-8 폴백

> **CDN 의존성 주의**: 파일 오픈 시 인터넷(esm.sh) 접근이 필요합니다. 행내 PC + 행내 와이파이 환경에서는 정상 작동하며, 오프라인 또는 인터넷 차단 환경에서는 빈 화면이 됩니다. 보고 전날 PDF 백업을 권장합니다.

## 3. 화면별 요구사항

### 3.1 탭 1 — 금리 조정 시뮬레이터 (`Simulator`)

#### 3.1.1 목적
전략등급(1~7등급)별로 금리(마진율)를 ±50bp 범위에서 조정했을 때, 신규유입·이탈률·잔액·수익성·신규고객 유입에 미치는 영향을 "현재" 대비 "조정 후" 값으로 즉시 비교한다.

> **등급 범위 정책**: 신규 승인 대상은 **1~7등급**만. 8~10등급 레거시 잔액(165.2억)은 총 잔액 카드에만 구분 표기.

#### 3.1.2 화면 구성 (상→하 순서)

##### (1) 헤더 영역
- Gauge 아이콘 + "전략등급 금리 조정 시뮬레이터"
- 버튼 3종: **최적금리 찾기(참고)** / **초기화** / **기준값 CSV** (Upload)

##### (2) 안내 박스
- 아이콘: `Info`(참고용 안내)
- 문구: "탄력성 계수: 등급별 **실측 다중회귀분석** 기반(2026-06)"
- 핵심 산식:
  - **공헌이익 = 이자수익 − 자금원가율 + 법적비용 + 계약신용원가** _(비용율 CSV 음수(−) 입력)_
  - **영업이익 = 공헌이익 − 계약업무원가 − 신용원가gap − 업무원가gap + 비이자이익** _(비용율 CSV 음수(−) 입력)_
- CSV 헤더 스펙: `grade,balance,count,newWk,netWk,churn,margin,`**`legalCostRate,contractCreditRate,contractOpRate,creditGapRate,opGapRate,nonIntRate`**`,elasticity,`**`newCustRatio`**
- 현재 적용 파일명 표시 (`현재: {baseFile}`)
- **자금원가율(cof) 입력**: 기본값 **2.8%**, 즉시 반영

##### (3) 등급별 조정 슬라이더
- 1~7등급, 각 행: 슬라이더(`min=-50, max=50, step=5bp`) + 조정값 표시 + `ε {elasticity}` 표시

##### (4) "신규유입 변화" 섹션 — 즉시효과 + +3M 잔액 _(v2.1 변경)_

| 라벨 | 단위 | 산출 | 비고 |
|---|---|---|---|
| 신규취급액 (주) | 억 | `newWkOld → newWkNew` (소수점 1자리) | 당주 즉시효과 |
| 순증액 (주) | 억 | `netWkOld → netWkNew` (소수점 1자리) | 당주 즉시효과 |
| **+3M 예상 대출잔액 (총합)** | 억 | `(balOld+165.2) → (balNew+165.2)` (소수점 1자리) | **v2.1**: `balOld/balNew`가 더 이상 "현재 잔액×이탈율 비율"이 아니라 **"현재 잔액 + 주간 순증액×12주"** 누적 방식. 전략 1~7등급 + 레거시 8~10등급 구분 2줄 표기 |
| 대출 건수 (주간) | 건 | `cntOld → cntNew` (정수, 콤마) | 당주 즉시효과 |
| **예상 신규고객 유입(주간)** | 명 | `custGainOld → custGain` (정수, 콤마) | 당주 즉시효과. 하단에 "3개월 평균 당행 최초 등록고객 비율 반영" 표시 |

- `custGainOld = Σ(cntOld × newCustRatio/100)` / `custGain = Σ(cntNew × newCustRatio/100)` (절대 수치, before/after 비교)
- 미니노트에 "대출 잔액(총합)은 +3개월 예상치, 그 외 카드는 당주 즉시효과"라는 구분 안내 추가

## 4. 계산 로직 정의

### 4.1 공통 유틸리티
| 함수 | 정의 |
|---|---|
| `avg(rows, key)` | 배열 평균 (빈 배열→0) |
| `sumBy(rows, key)` | 배열 합계 |
| `toMonthlyMM(v)` | `v × 100 / 12` — 연간(억) → 월간(백만원) |

### 4.2 `computeRows(adjMap, base, cofVal)` — 등급별 시뮬레이션 _(v2.1 산식 변경)_

> **부호 규칙**: `legalCostRate`, `contractCreditRate`, `contractOpRate`, `creditGapRate`, `opGapRate` 5종은 CSV에서 음수(−) 입력. `nonIntRate`·`newCustRatio`는 양수(+) 입력.

```
churnNew    = max(0, churn + (bp/10) × elasticity)
retainRatio = (100 − churnNew) / (100 − churn)
netWkNew    = netWk × retainRatio

// +3개월(12주) 예상 잔액 — v2.1: 이탈율 비율 스케일링 → 주간 순증액 누적 방식으로 전환
balOld      = balance + netWk    × 12   // 조정 없이 현재 추세 유지 시
balNew      = balance + netWkNew × 12   // 조정 후 추세 반영 시

marginNew   = margin + bp/100
intOld      = balOld × margin   / 100                       // +3M 시점 연간 이자수익(억원)
intNew      = balNew × marginNew / 100

// 음수 입력값 → + 처리로 차감 효과 동일
cmRateOld   = margin    − cof + legalCostRate + contractCreditRate
cmRateNew   = marginNew − cof + legalCostRate + contractCreditRate
cmOld/New   = balOld/balNew × cmRate / 100                   // +3M 시점 연간 공헌이익(억원)

// 음수 입력값 → − 처리로 양수 비용 전환
opAdjRate   = −contractOpRate − creditGapRate − opGapRate − nonIntRate
opOld/New   = cmOld/New − balOld/balNew × opAdjRate / 100    // +3M 시점 연간 영업이익(억원)

cntNew = count × retainRatio  /  newWkNew = newWk × retainRatio
newCustRatio: 해당 등급 실측 신규고객 비율 — computeRows 반환값에 포함
```

> **왜 바꿨나(배경)**: 기존 "이탈율 변화 비율 × 현재 잔액" 방식은 "균형점에 도달하는 시점이 언제인지 불명확"하다는 약점이 있었습니다. 사용자 제안으로, 조정 후 예상되는 **주간 순증액이 12주(약 3개월)간 유지된다고 가정하고 흐름을 누적**하는 방식으로 전환해 "+3개월 후"라는 구체적 시점의 추정치로 재정의했습니다. `balOld`도 동일한 논리로 "조정 없이 현재 추세가 유지될 때의 +3개월 후 예상치"로 재정의되어, `bp=0`일 때 `balOld=balNew`(변화 없음)인 기존 before/after 패턴이 그대로 유지됩니다.

### 4.3 `runSimulation(adjMap, base, cofVal, indirectParams)`

```
// 신규고객 유입 수 (절대값)
custGainOld = Σ (r.cntOld × r.newCustRatio / 100)   // 현재 주간 신규고객 수
custGain    = Σ (r.cntNew × r.newCustRatio / 100)   // 조정 후 주간 신규고객 수

// 간접효과 — custGain(조정 후 절대 수치) 기준
cardEffect    = custGain × (cardRate / 100)
depositEffect = custGain × depositRate / 10000      // 억; 표시 시 ×100 → 백만원
mauEffect     = custGain × (mauRate / 100)

반환: { rows, tot, indirect: { custGainOld, custGain, cardEffect, depositEffect, mauEffect } }
```

### 4.4 `findOptimalAdj(base, cofVal, { churnCap=15, range=50, step=5 })` _(v2.1 산식 변경)_
- 등급별 grid search: 공헌이익 최대화 bp, 이탈률 증가 ≤ 15%p 제약
- balNew 산식도 §4.2와 동일하게 전환: `balNew = balance + (netWk × retainRatio) × 12`
- cm 산식: `balNew × (margin + bp/100 − cof + legalCostRate + contractCreditRate) / 100`

### 4.5 ResultPage 집계
- 주간 KPI: `avg(before/after, key)` → delta%
- 월간 KPI: `avg(filtered by beforeMonths/afterMonths, key) × 100` (억→백만원)
- 통합 차트: 고신용 + 중저신용 합산 × 100

### 4.6 `LEGACY_BALANCE` 상수
```js
const LEGACY_BALANCE = 165.2; // 8~10등급 레거시 잔액(억) — 조정 범위 외
```
- 적용: Simulator **"+3M 예상 대출잔액 (총합)"** 카드 단 한 곳
- 시뮬레이션 반영 안 됨 (취급액/수익성 계산은 1~7등급만)
- 변경 시 HTML 내 상수 직접 수정

### 4.7 `weeklyAxisMax` — 주간 차트 공통 Y축 상한 (ResultPage)
```
vals          = 모든 이벤트(weeklyByEvent: ev331+ev423+ev529)의 newLoan, netLoan 값 전체
weeklyAxisMax = ceil( max(vals) × 1.15 / 10 ) × 10
```
적용 위치: ResultPage 주간 차트(§3.2.3 (5))의 `<YAxis domain={[0, weeklyAxisMax]} />`.

### 4.8 `elasticity`(탄력성 계수) 추정 방법론 — 요약

**모형** (등급별 개별 OLS 회귀, 1~4등급만):
```
churn_i,t = β0 + β1·bp_i,t + β2·ret_t + ε_i,t
elasticity = β1 × 10
```
**최종 결과**: 1등급=2.15, 2등급=1.44, 3등급=0.50, 4등급=0.91 (10bp당, R²=0.08~0.19). 상세 진단(다중공선성 발견·보정 과정)은 [`CSV_데이터_가이드.md`](./CSV_데이터_가이드.md) 부록 참고.
