## 1. 시스템 개요

### 1.1 목적
사장님신용대출의 **전략등급별 금리(마진율)를 조정했을 때** 발생하는 다음 효과를 한 화면에서 시뮬레이션하고,
실제 금리 변경 시행 후의 **실적 변화를 추적**하기 위한 대시보드.

1. 신규유입 변화(즉시효과): 신규취급액, 순증액, 대출잔액(전체), 대출건수, 신규고객 증가분
2. 손익 변화(균형추정 — 월간 기준): 이자수익, 공헌이익, 영업이익
3. 간접효과: 카드발급고객, 연동 예금평잔, MAU 고객 증가
4. 시행 실적: 과거 3건의 금리 변경 이벤트(3/31, 4/23, 5/29)에 대한 전/후 비교 (주간 영업지표, 월간 수익성, 퍼널 전환율)

### 1.2 기술 스택 / 구현 방식
- **단일 HTML 파일**로 구현 (배포·공유 용이성 우선, 별도 빌드 과정 없음)
- `importmap` + esm.sh CDN을 통해 React 18.3.1 / ReactDOM 18.3.1 / Recharts 2.12.7 / lucide-react 0.383.0 로드
- `<script type="text/babel" data-presets="react" data-type="module">` — Babel Standalone으로 브라우저 내 JSX 인라인 트랜스파일 (오프라인 환경에서는 CDN 접근 필요)
- 차트: Recharts (`BarChart`, `Bar`, `LabelList`, `ResponsiveContainer`, `Tooltip`, `Legend`, `ReferenceLine`, `CartesianGrid`, `XAxis`, `YAxis`)
- 아이콘: lucide-react (`Sliders`, `BarChart3`, `Gauge`, `Wand2`, `Upload`, `AlertTriangle`, `Info`, `TrendingUp`, `TrendingDown`, `Users`, `PiggyBank`, `Activity` 등)
- 상태관리: React `useState` / `useMemo` (전역 상태 라이브러리 없음, `App` 컴포넌트에 전체 상태 보유 후 props로 하위 전달)
- 데이터 입력: 모든 외부 데이터는 **CSV 파일 업로드**를 통해 클라이언트 사이드에서 파싱(FileReader) — 서버/백엔드 없음

## 3. 화면별 요구사항

## 3.1 탭 1 — 금리 조정 시뮬레이터 (`Simulator`)

### 3.1.1 목적
전략등급(1~7등급)별로 금리(마진율)를 ±50bp 범위에서 조정했을 때, 신규유입·이탈률·잔액·수익성에 미치는 영향을
"현재" 대비 "조정 후" 값으로 즉시 비교한다.

> **등급 범위 정책(v0.3 명시)**: 현재 신규 승인 대상은 **1~7등급**만 해당. 과거 승인된 8~10등급 잔액(레거시)은 존재하나 신규 유입·조정 시뮬레이션 대상에서 제외. 단, 총 대출잔액 카드에는 레거시 잔액을 구분 표기하여 전체 포트폴리오 규모를 정확히 보고.

### 3.1.2 화면 구성 (상→하 순서)

#### (1) 헤더 영역 (`S.simHeadRow`)
- 좌측: Gauge 아이콘 + "전략등급 금리 조정 시뮬레이터" 제목
- 우측 액션 버튼 3종:
  - **최적금리 찾기(참고)** (Wand2 아이콘) — `findOptimalAdj` 실행, 등급별 공헌이익을 최대화하는 bp 조정값을 자동 산출하여 슬라이더에 반영
  - **초기화** — 모든 등급의 조정값(bp)을 0으로 리셋
  - **기준값 CSV** (Upload 아이콘) — `template_2_simbase.csv` 형식 파일 업로드 → 시뮬레이션 기준값(`simBase`) 전체 교체 + 조정값 초기화

#### (2) 가정값 안내 박스 (`S.assumptionBox`)
- AlertTriangle 아이콘
- 문구: "탄력성 계수 및 원가율 6종은 등급별 **가정값**이며 실측치 확보 시 기준값 CSV로 교체하세요."
- 핵심 산식 노출:
  - **공헌이익 = 이자수익 − 자금원가율 − 법적비용 − 계약신용원가**
  - **영업이익 = 공헌이익 − 계약업무원가 − 신용원가gap − 업무원가gap + 비이자이익**
- 기준값 CSV 헤더 스펙 텍스트: `grade,balance,count,newWk,netWk,churn,margin,legalCostRate,contractCreditRate,contractOpRate,creditGapRate,opGapRate,nonIntRate,elasticity`
- 현재 적용 중인 기준값 파일명 표시 (`현재: {baseFile}` — 기본값 "기본 가정값")
- **자금원가율(cof) 입력 필드**: "(사장님 신용대출 3개월 평균) 자금원가율 [입력칸]%" — 사용자가 직접 입력 (기본값 1.7%), 모든 공헌이익/영업이익 계산에 즉시 반영

#### (3) 등급별 조정 슬라이더 (`S.sliderWrap`)
- 1등급~7등급, 각 행마다:
  - 등급명 표시
  - 슬라이더: `min=-50, max=50, step=5` (bp 단위)
  - 현재 조정값 표시: `{+/-}{bp}bp` (양수=`primary` 색, 음수=`secondary` 색, 0=`faint` 색)
  - 탄력성 계수 표시: `ε {elasticity}`

#### (4) "신규유입 변화" 섹션 — 즉시효과 (`S.subSectionLabel`)
- 부제: "(즉시효과 — 금리 조정 시 단기 반응)"
- 카드 5종 (`volTotals`, 커스텀 `renderCard`로 렌더):

| 라벨 | 단위 | 산출 | 비고 |
|---|---|---|---|
| 신규취급액 (주) | 억 | `sim.tot.newWkOld → newWkNew` (소수점 1자리) | — |
| 순증액 (주) | 억 | `sim.tot.netWkOld → netWkNew` (소수점 1자리) | — |
| **대출 잔액 (총합)** | 억 | `(sim.tot.balOld + LEGACY_BALANCE) → (sim.tot.balNew + LEGACY_BALANCE)` (소수점 1자리, 총합 표시) | **v0.3 신규**: 카드 내 2줄 구분 표기 |
| 대출 건수 (주간) | 건 | `sim.tot.cntOld → cntNew` (정수, 콤마) | — |
| 신규고객 증가분 | 명 | `sim.indirect.custGain` — 증감분만 표시(`noBase: true`) | — |

> **v0.3 신규 — 대출 잔액 카드 상세 구조**:
>
> 카드 상단 주요 수치에는 **1~10등급 총합** `(sim.tot.balNew + LEGACY_BALANCE).toFixed(1)억`을 표시합니다.
> 카드 하단에는 2줄로 구분하여 표기합니다:
> - **전략 1~7등급** `{sim.tot.balNew.toFixed(1)}억` (조정에 따른 증감 표시, `up`/`down` 색)
> - **레거시 8~10등급** `165.2억` · **조정 외** (`faint` 색, 고정값)
>
> 이를 통해 CEO/ALCO 보고 시 "총 포트폴리오 규모"와 "전략 조정 대상 포트폴리오" 및 "레거시 잔존 잔액"을 한 카드에서 구분 확인 가능합니다.

## 4. 계산 로직 정의

### 4.1 공통 유틸리티
| 함수 | 정의 |
|---|---|
| `avg(rows, key)` | `rows`의 `key` 값 평균 (빈 배열이면 0) |
| `sumBy(rows, key)` | `rows`의 `key` 값 합계 |
| `toMonthlyMM(v)` | `v × 100 / 12` — 연간(억원) → 월간(백만원) 환산 |

### 4.2 `computeRows(adjMap, base, cofVal)` — 등급별 시뮬레이션
입력: 등급별 조정값 맵(`adjMap`, bp 단위), 기준값 배열(`base` = simBase 1~7등급), 자금원가율(`cofVal`, %)

```
churnNew    = max(0, churn + (bp/10) × elasticity)
retainRatio = (100 − churnNew) / (100 − churn)
balNew      = balance × retainRatio

marginNew   = margin + bp/100
intOld      = balance × margin   / 100
intNew      = balNew  × marginNew / 100

cmRateOld   = margin    − cof − legalCostRate − contractCreditRate
cmRateNew   = marginNew − cof − legalCostRate − contractCreditRate
cmOld       = balance × cmRateOld / 100
cmNew       = balNew  × cmRateNew / 100

opAdjRate   = contractOpRate + creditGapRate + opGapRate − nonIntRate
opOld       = cmOld − balance × opAdjRate / 100
opNew       = cmNew − balNew  × opAdjRate / 100

cntNew = count × retainRatio  /  newWkNew = newWk × retainRatio  /  netWkNew = netWk × retainRatio
```

### 4.3 `runSimulation(adjMap, base, cofVal, indirectParams)`
- `computeRows()` 결과를 `sumBy()`로 합산 → `tot` 객체 (balOld/New, cntOld/New 등)
- `custGain = tot.cntNew − tot.cntOld`
- 간접효과: `cardEffect`, `depositEffect`(억, 표시 시 ×100=백만원), `mauEffect`
- 반환: `{ rows, tot, indirect: { custGain, cardEffect, depositEffect, mauEffect } }`

### 4.4 `findOptimalAdj(base, cofVal, { churnCap=15, range=50, step=5 })`
- 등급별 grid search: 공헌이익 최대화 bp 탐색, 이탈률 증가폭 ≤ 15%p 제약
- 반환: 등급별 최적 bp 맵 → "최적금리 찾기" 버튼 적용

### 4.5 ResultPage 집계 로직
- **주간 KPI**: `avg(weeklyBefore/After, key)` → delta%
- **월간 KPI**: `avg(monthlyHigh/Low filtered by beforeMonths/afterMonths, key)` → delta%
- **통합 월간 차트**: `(고신용.metric + 중저신용.metric) × 100` per month

### 4.6 `LEGACY_BALANCE` 상수 — 레거시 잔액 관리 (v0.3 신설)

```js
const LEGACY_BALANCE = 165.2; // 8~10등급 레거시 잔액(억) — 현 전략 조정 범위 외
```

| 항목 | 내용 |
|---|---|
| 값 | **165.2억원** (2026-06-18 기준 실잔액 업데이트) |
| 적용 위치 | Simulator "대출 잔액 (총합)" 카드 **단 한 곳**만 |
| 카드 표시 | 총합(1~10등급) → 하단에 "전략 1~7등급 {balNew}억 (±{d}억)" + "레거시 8~10등급 165.2억 · 조정 외" |
| 시뮬레이션 반영 여부 | **반영 안 됨** — 조정 효과(취급액/순증액/수익성)는 1~7등급만 |
| 값 갱신 방법 | 잔액 변동 시 `standalone.html` 내 `const LEGACY_BALANCE = 165.2;` 직접 수정 |
| CSV 연동 여부 | **없음** — CSV 템플릿 외부에서 하드코딩 관리 |
| 향후 고도화 경로 | 8~10등급 잔액 소진 시 상수를 0으로 수정하거나 해당 코드 제거. 별도 전략 수립 시 `simBase`에 등급 추가 + 슬라이더 확장 방식으로 전환 |

## 5. CSV 데이터 입력 명세

> 전체 CSV 템플릿의 컬럼별 정의·단위·산식·데이터 추출 가이드는 [`CSV_데이터_가이드.md`](./CSV_데이터_가이드.md)에 상세 기술.

| CSV 파일 | 업로드 위치 | 적용 범위 | 파서 함수 |
|---|---|---|---|
| `template_2_simbase.csv` | 시뮬레이터 탭 상단 "기준값 CSV" | **1~7등급만** 시뮬레이션 기준값 | `parseSimBase` |
| `template_1_weekly_ev331/423/529.csv` | 시행 실적 탭, 이벤트별 "주간 실적 CSV" | 해당 이벤트의 주간 영업지표 (`balance` 컬럼 = 1~10등급 전체 잔액 권장) | `parseWeekly` |
| `template_3_monthly_high.csv` | 헤더 우측 "고신용 수익성 CSV (월간)" | 전체 이벤트 공유 (고신용 1~6월) | `parseMonthly` |
| `template_3_monthly_low.csv` | 헤더 우측 "중저신용 수익성 CSV (월간)" | 전체 이벤트 공유 (중저신용 1~6월) | `parseMonthly` |
| `template_4_funnel_ev331/423/529.csv` | 시행 실적 탭, 이벤트별 "퍼널 CSV" | 해당 이벤트의 퍼널 전환율 | `parseFunnelWide` |

- `template_2_simbase.csv`는 **1~7등급만** 포함 (8~10등급 레거시는 `LEGACY_BALANCE` 상수로 별도 관리)
- `template_1_weekly_ev*.csv`의 `balance` 컬럼은 실측 데이터 입력 시 **1~10등급 전체 잔액**을 기입하는 것이 권장됩니다 (레거시 포함 총 포트폴리오 추이 파악용)
- v0.2 이후 억원 단위 물량 컬럼(`balance`, `newWk`/`netWk`, `newLoan`/`netLoan`)은 소수점 1자리까지 표시되므로 데이터도 소수 1자리 입력 권장

## 7. 미결정 / 추후 협의 항목 (Open Items)

1. **ADW 데이터 연동**: 수익성 지표 ADW 자동 연동 예정 (현재 수동 CSV 교체).
2. **탄력성 계수(`elasticity`) 및 원가율 6종**: 현재 가정값. ADW 실측 데이터 확보 후 역산하여 교체 필요.
3. **자금원가율(cof)**: 현재 직접 입력(기본값 1.7%). 자동 연동 방식 전환 검토.
4. **간접효과 전환율(n%)**: 현재 가정값(35%/120만원/60%). 실측 확정 시 대체.
5. **시뮬레이터 서브섹션 캡션 검증**: "신규유입 변화(즉시효과)" / "추정 손익(균형추정)" 캡션 원래 의도 최종 확인 필요.
6. **고신용/중저신용 분류 기준 문서화**: 신용평가 등급 컷오프 기준 명확화 필요.
7. **이벤트(EVENTS) 확장성**: 신규 금리 변경 이벤트 추가 시 `EVENTS` + CSV 3종 + `beforeMonths`/`afterMonths` 동시 갱신 필요.
8. **백업 갱신**: `standalone(backup0612).html`이 최신 변경(2026-06-13~16) 미반영 상태. 현재 기준 백업 생성 여부 확인.
9. **연동 예금평잔 단위 적절성**: 억→백만원 변경으로 자릿수 증가. 보고용 적절성 실데이터 입력 후 재검토.
10. **`LEGACY_BALANCE` 갱신 프로세스**: 현재 하드코딩(165.2억). 레거시 잔액 변동 시 담당자가 HTML을 직접 수정해야 함. 향후 실잔액 연동 방식(CSV 또는 ADW) 고도화 여부 협의. 8~10등급 잔액이 소진되면 해당 상수 제거 및 카드 표시 로직도 정리 필요.
